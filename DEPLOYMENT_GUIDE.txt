# GTTC Result Management System - Full Deployment Guide

## 1. Prerequisites
- A GitHub account with the project repository.
- A Railway account (recommended for full-stack Node.js apps).
- A Google Gemini API Key (from [Google AI Studio](https://aistudio.google.com/)).

## 2. Deployment on Railway (Recommended)
Railway is the best choice for this app because it supports persistent storage for the SQLite database and runs the Express server seamlessly.

### Steps:
1. **Login to Railway** and click "New Project".
2. **Connect your GitHub Repository**.
3. **Add Environment Variables**:
   Go to the "Variables" tab in your Railway service and add:
   - `PORT`: `3000`
   - `JWT_SECRET`: `gen-lang-client-0765674254`
   - `GEMINI_API_KEY`: `your_actual_gemini_api_key`
   - `DATABASE_PATH`: `/data/gttc_results.db`
   - `NODE_ENV`: `production`

4. **Persistent Storage (Crucial for SQLite)**:
   By default, Railway's file system is ephemeral.
   - Go to your Project Dashboard -> **+ New** -> **Volume**.
   - Once created, go to your Service -> **Settings** -> **Volumes**.
   - Mount the volume at: `/data` (This matches the `DATABASE_PATH` above).
5. **Build & Start Commands**:
   Railway should automatically detect the `package.json`.
   - Build Command: `npm run build`
   - Start Command: `npm start`

## 4. Troubleshooting
- **"Error deploying from source"**: This usually means the build failed. Ensure `npm run build` (which runs `tsc`) completes without errors. Check the "Build" logs in Railway.
- **"API route not found: GET /api/auth/login"**: This happens if you try to visit the login API directly in your browser. The login route only accepts POST requests from the login form.
- **"Root Directory"**: Ensure the Root Directory in Railway's Build settings is set to `/`.
- **Data disappearing**: Ensure you have a "Volume" mounted in Railway if using SQLite.
- **Gemini errors**: Ensure your API key is valid and has access to `gemini-1.5-flash`.
